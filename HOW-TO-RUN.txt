HOW TO RUN — Tumble Trainer
===========================

The app must be served over HTTP. Opening index.html directly (file://)
will NOT work — the browser blocks the routine-seed.json fetch and the
service worker, and you'll see an init error page instead of the app.

Quick start
-----------
1. Open a terminal in this folder (h:\Projects\Workout App).
2. Start a local web server. Any of these works:

     python -m http.server 8080
     npx serve -l 8080
     php -S localhost:8080

3. Open http://localhost:8080 in your browser.

On your phone — installed, offline, no PC required
--------------------------------------------------
The app is deployed to GitHub Pages:

    https://mattdchilds.github.io/workout-app/

1. Open that URL on the phone in Chrome (Android) or Safari (iOS).
2. Let it finish loading once, on a network. The service worker
   precaches every asset on that first visit.
3. Install it:
     Android/Chrome — menu > "Install app". If you only see
       "Add to Home Screen", the manifest didn't validate; the
       install is then a bookmark, not a standalone PWA.
     iOS/Safari     — Share > "Add to Home Screen".
4. Done. Turn on airplane mode, power off this PC — it still works.
   Only the Ask tab needs a network (it calls the Anthropic API).

Deploying an update: commit and `git push`. Pages rebuilds in ~1 min.
Bump CACHE in sw.js or phones will keep serving the old cached copy.

On your phone via LAN (dev only — NOT installable)
--------------------------------------------------
Useful for testing a change without pushing, but note that a service
worker only registers in a secure context (https:// or localhost). Over
a LAN IP the registration in app.js fails, is swallowed by its .catch(),
and the app runs as a plain web page: no offline, no real install.

1. Start the server as above.
2. Find this PC's local IP:  ipconfig  (look for "IPv4 Address").
3. On the phone, browse to http://<that-ip>:8080
   Windows Firewall must allow inbound python.exe on the active
   network profile, or the phone will just time out.

If changes don't show up
------------------------
The service worker caches everything for offline use. After code or
data changes:
  - The cache name in sw.js (CACHE = 'tumble-trainer-vX.Y.Z') must be
    bumped — this is already done for each release.
  - Then hard-refresh (Ctrl+Shift+R), or refresh twice: the first load
    installs the new worker, the second serves the new files.
  - Nuclear option: DevTools > Application > Service Workers >
    Unregister, then Clear storage > Clear site data (NOTE: clearing
    site data also wipes your saved routine/log in localStorage —
    export a backup first from the Settings tab).

Your data
---------
Everything (routine, session log, progression levels, API key) lives in
this browser's localStorage under the key "tumbleTrainer.v2". Use the
export/import options in the Settings tab to back up or move devices.

Dev notes
---------
- No build step, no dependencies: plain HTML/CSS/JS.
- Headless smoke checks: node --check app.js  (app.js also exposes a
  module.exports hook so pure functions can be required from Node).
